🔴 问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  Failed to execute 'setItem' on 'Storage': 
  Setting the value of 'image_history' exceeded the quota.

  原因: Flux 模型返回 ~2-3 MB Base64 数据
       localStorage 限制 5-10 MB
       保存多张图片会超限

✅ 解决方案
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  1. 创建 StorageService (IndexedDB)
     - 存储容量: 50+ MB
     - 支持异步操作
     - 自动降级到 localStorage

  2. 更新 ImageGeneratorNew.tsx
     - 使用 StorageService 保存历史
     - 处理异步操作
     - 改进错误处理

  3. 改进 ImageGenerator.tsx
     - Base64 → Blob 下载
     - 支持大型数据

🔧 核心功能
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  StorageService.saveImageToHistory(url)
    → 保存图像到 IndexedDB
    → 自动限制 20 张
    → 降级到 localStorage

  StorageService.getImageHistory(limit)
    → 获取历史记录
    → 按时间戳排序
    → 返回最近的 N 张

  StorageService.clearImageHistory()
    → 清空所有历史记录
